# Week 1 Growth Plan

## Goal
Turn the newly published repository into a page that converts cold visitors into stars within 10–30 seconds.

## Current status
- Public repo is live
- `v0.1.0` release is live
- Core publish surface exists
- Next priority is conversion optimization + first-wave distribution

## Priority order

### 1. Tighten the GitHub repo surface
- Verify About / Description / Topics are fully applied
- Keep the README hero tight and proof-driven
- Add a minimal shell screenshot later if it materially improves portability proof
- Check desktop + mobile readability on the live GitHub page

### 2. Ship the first launch wave
- Chinese launch post first
- English launch post second
- Choose Reddit or Hacker News for the third move, not both at once
- Use the release link as a trust anchor where helpful

### 3. Start measurement immediately
- Record stars at repo publish
- Record stars after release
- Record stars after each external post
- Note what framing gets the best reactions

### 4. Use feedback to iterate fast
- Clarify anything people misunderstand
- Promote the strongest angle harder
- Cut wording that does not increase comprehension or star intent

## First-wave channel recommendation
1. GitHub release (done)
2. Chinese launch post
3. English launch post
4. Reddit or Hacker News (pick one first)

## Decision rule
Do not spend week 1 polishing low-value internals. Improve only what changes:
- click-through to repo
- 10-second comprehension
- trust
- star intent
